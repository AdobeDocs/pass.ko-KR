# 동시성 모니터링 시작 {#getting-started-overview}

동시 모니터링 시작! 이 안내서는 기본 사항을 이해하고 통합을 빠르게 시작하고 실행하는 데 도움이 됩니다.

## 동시성 모니터링으로 해결할 수 있는 문제 {#problem-solved}

오늘날의 스트리밍 환경에서는 구독자가 여러 디바이스와 애플리케이션에서 콘텐츠에 액세스할 것으로 예상합니다. 그러나 이로 인해 콘텐츠 공급자에게 문제가 발생합니다.

- **콘텐츠 라이선스 계약**&#x200B;은(는) 종종 동시 스트림 수를 제한합니다.
- 과도한 동시 사용으로 **대역폭 비용** 증가
- 너무 많은 스트림이 성능을 저하시키면 **사용자 경험**&#x200B;이(가) 발생합니다.
- **수익 보호**&#x200B;를 사용하려면 계정 공유 남용을 방지해야 합니다.

동시성 모니터링은 이러한 문제에 대한 중앙 집중식 솔루션을 제공합니다.


## 동시성 모니터링 작동 방식

### 핵심 기능

동시성 모니터링은 다음과 같은 **세션 관리 서비스**(으)로 작동합니다.

1. 모든 응용 프로그램에서 실시간으로 **활성 스트리밍 세션을 추적합니다**
2. 현재 사용 패턴에 대해 **비즈니스 정책 평가**
3. 정책을 위반할 때 **제한 적용**
4. 충돌이 발생할 때 **해결 옵션을 제공합니다**

## 다양한 이해 관계자를 위한 이점

### 콘텐츠 공급자(프로그래머)

- **콘텐츠 권한 보호** 및 라이선스 계약 준수
- 과도한 사용을 방지하여 **대역폭 비용 최적화**
- 제한에 대한 명확한 피드백을 통해 **사용자 경험 개선**
- 자세한 보고를 통해 **사용 인사이트 확보**

### ID 공급자(MVPD)

- 모든 콘텐츠 파트너의 **구독자 계약 적용**
- 여러 응용 프로그램에 대한 **중앙 집중식 정책 관리**
- 사용 패턴의 **실시간 모니터링**
- 수백만 개의 세션을 처리하는 **확장 가능한 아키텍처**

### 최종 사용자

- 사용 제한 **이해 지우기**
- 모든 응용 프로그램에서 **일관된 경험**
- 한도에 도달하면 **충돌을 해결하는 옵션**
- 액세스가 제한되는 이유에 대한 **투명한 피드백**

## 서비스 등록 {#service-registration}

동시성 모니터링을 시작하려면 [지원 팀](mailto:tve-support@adobe.com)에 다음 정보를 문의하세요.

1. **회사 이름** 및 연락처 세부 정보
2. 동시성 모니터링과 통합하려는 **응용 프로그램**. 각 애플리케이션에 대해 다음을 입력하십시오.
   1. 애플리케이션 이름
   2. 애플리케이션 플랫폼
3. **통합 파트너**(다른 파티, 프로그래머 또는 MVPD의 요청으로 동시 모니터링 구독 중인 경우)
